## What changed
Trellis grows a third rule alongside `guidelines.md` and `language.md`. Where `guidelines.md`
covers how to write and ship and `language.md` covers public-facing voice, `conventions.md`
governs the shape of the code itself.

Its first rule: version every API in the URL path (`/v1/`, `/v2/`), reserving a new number for a
breaking change - not a query parameter, not a header. Each rule is a single, checkable line, so
more conventions can land here over time without turning it into an essay.

The rule ships in both the installed copy (`docs/rules/`) and the source (`trellis/rules/`), is
marked Trellis-owned so `/trellis-update` keeps it current, and the README, CONTRIBUTING,
`AGENTS.md`, and the features overview all point at it.
