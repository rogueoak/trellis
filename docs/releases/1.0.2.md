`conventions.md` grows from one rule to four: a nesting-depth cap, no ternaries, and design-system-token styling.

## What changed
The `conventions.md` rule - how the code itself is written - picks up three new conventions
alongside URL-path API versioning:

- **Control flow.** Never nest logic more than 4 levels deep. `if`/`else`, loops, lambdas, and
  callbacks each add a level; past 4, extract a function or invert with an early return.
- **Readability.** Avoid ternary operators, especially in JSX - they are hard to read. Hoist the
  logic above the JSX block and assign the result to a variable or component, then render that.
- **Styling.** Style with the design system tokens. Do not invent custom styling; if you need a
  style the design system does not support, pause and ask about adding it rather than hardcoding a
  one-off value.

Each rule stays a single, checkable line with a Yes/No example. The installed copy
(`docs/rules/`) and the source (`trellis/rules/`) stay identical, and the README rules table and
features overview now cover the broader set.
